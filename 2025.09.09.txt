1. 전용회선 : 항상 연결됨 (고정 연결), 고속, 안정성 높음, 비용이 비쌈
 1) T1, E1, T3, STM-1 등
 2) 광랜 전용회선, KT 전용선, SK B2B 회선
 3) 프레임 릴레이(Frame Relay) → 전용 접속 방식으로 사용 가능
 4) Metro Ethernet

2. 회선교환망 : 통화나 연결 동안 전용 경로 유지, 실시간 음성통신에 적합, 사용 중일 때만 연결됨
 1) 전화망(PSTN, Public Switched Telephone Network)
 2) ISDN (Integrated Services Digital Network)

 4) 2G GSM 음성 통화

3. 패킷교환망 : 경로 고정 안 됨, 실시간성은 낮지만 효율적, 자원 공유, 네트워크 효율 높음
 1) 인터넷 (IP 기반)
 2) X.25
 3) 프레임 릴레이 (Frame Relay) (동시에 전용 접속망으로도 사용됨)

- 라우터 프로토콜
1. HDLC (High-Level Data Link Control): ISO에서 정의한 데이터링크 계층 프로토콜
2. Cisco HDLC: Cisco가 독자적으로 확장한 HDLC 구현 (표준과 다름 – 벤더 종속적)
3. 이기종 장비 환경이나 인터넷 회선을 사용할 경우, HDLC보다는 PPP를 사용하는 경우가 훨씬 많다
  - 하지만 그 사용처는 제한적이며, 과거보다 사용 빈도는 현저히 줄어든 상태
4. PPP와 HDLC는 라우터가 사용하는 링크 계층(2계층)의 WAN 인캡슐레이션 프로토콜이다. 즉 라우터끼리(또는 다른 WAN 장비 간) 통신을 위해 사용하는 데이터링크 계층 프로토콜
 1) PPP와 HDLC는 "점대점(Point-to-Point)" 연결에서 사용되는 인캡슐레이션 프로토콜이다.
 2) 이런 직렬 링크(WAN 링크) 상에서 라우터끼리 데이터를 주고받기 위해 사용된다.
 3) 즉, 라우터 ↔ 라우터 통신을 가능하게 하는 기본 틀 중 하나다.

- 스위치 네트워크 구성(NETWORK DESIGN & CONFIGURATION)

1. Distribution Switch(규모에따라 이중화) 와 Access Switch(여러대)로 이루어진 네트워크


2. Campus Network Design(3-Tier)
- R&E Network : "Research and Education Network"의 약자로, 주로 연구 기관, 대학, 교육 기관을 연결하는 고성능 데이터 네트워크
- 3-Tier 네트워크 아키텍처는 기업 네트워크를 세 가지 계층으로 구성하여 확장성과 관리 효율성을 높이는 구조(Core, Distribution, Access)

1) Core Switch : 코어 스위치는 네트워크의 중심에서 고속 데이터 처리를 담당하는 고성능 스위치(건물과 건물등)
  - 제품 : Cisco Catalyst 9000/Cisco Nexus 9000
  - Fast Transport, No Layer 3 Processing(빠르게 데이터를 전달하기 위해 L3작업을 하지 않는다.)
  - 네트워크의 고속 백본(backbone) 역할을 담당
  - 네트워크 트래픽의 주요 경로를 제공하고, 데이터가 네트워크의 다른 부분(예: 액세스 레이어, 배포 레이어)으로 원활하게 전송될 수 있도록 고성능의 스위칭 기능을 제공

2) Distribution Switch : 디스트리뷰션 스위치(Distribution Switch)는 네트워크의 분산 계층에 위치하여 코어 스위치와 액세스 스위치 사이에서 트래픽을 집계하고, 라우팅, 보안, 정책 적용을 담당
 - Cisco Catalyst 9300/Cisco Catalyst 9400
 - 3계층 장비
 - Access Aggregation Point, Workgroup Services Access, Broadcast Domain Definition, Inter-VLAN Roution, Media Translation, Security

3) Access Switch : 액세스 스위치(Access Switch)는 네트워크의 액세스 계층에 위치하여 사용자 디바이스(컴퓨터, IP 전화, 무선 액세스 포인트 등)를 네트워크에 연결하는 역할을 담당
 - Cisco Catalyst 9200/Cisco Catalyst 2960
 - Layer 2 Services (Filtering, VLAN), Shared Bandwidth

1. 전이중 통신 (Full-Duplex) : 양방향 통신을 동시에 가능하게 하는 방식
 1) 전화 통화, 유선 LAN(Ethernet, 스위치 연결 시), 광케이블
 2) 송신과 수신을 동시에 할 수 있어 속도 빠름, 충돌 없음(두 개의 채널 사용 또는 양방향 회선 구성 (Tx/Rx 따로))

2. 반이중 통신 (Half-Duplex) : 양방향 통신이 가능하지만, 동시에는 불가능한 방식 (한 번에 한 방향만 통신)
 1) 무전기, 구형 허브 기반 이더넷(10Base2/5), Walkie-Talkie
 2) 송신 중에는 수신 불가 → 충돌 가능성 있음, CSMA/CD 사용(하나의 채널을 양방향 공유)



CSMA/CD(Carrier Sense Multiple Access with Collision Detection) --->  ethernet 방식(LAN)
네트워크 통신에서 사용되는 접근 제어 방식으로,
Carrier Sense (전송 감지): 먼저 네트워크가 사용 중인지 확인
Multiple Access (다중 접근): 여러 장치가 동시에 네트워크에 접근 가능
Collision Detection (충돌 감지): 동시에 전송 시 충돌을 감지하고, 충돌 발생 시 재전송


1. ARP(IP 주소를 MAC 주소와 매칭시키기 위한 프로토콜)  ip 주소를 이용해서 mac 주소를 알 수 있다.  (RARP: mac주소로 ip주소 알수있다.)
주소 결정 프로토콜(Address Resolution Protocol, ARP)은 네트워크 상에서 IP 주소를 물리적 네트워크 주소로 대응(bind)시키기 위해 사용되는 프로토콜이다. 
여기서 물리적 네트워크 주소는 이더넷 또는 토큰링의 48 비트 네트워크 카드 주소를 뜻한다.

2. MAC Address(Media Access Control Address) 는 네트워크 인터페이스 카드( NIC )에 할당된 고유 ID이다. 
물리적 주소 또는 하드웨어 주소 라고도 한다. 하드웨어 제조업체를 식별하고 네트워크 세그먼트의 장치 간 네트워크 통신에 사용된다. MAC 주소 는 일반적으로 2개의 16진수로 구성된 6개의 그룹( 6 octets )으로 구성된다 ( 총 48bit )
- 공급업체 OUI(Organizationally Unique Identifier)

- 소켓 작동 원리
(1) 소켓 생성 : 통신을 시작하기 위해 소켓을 생성한다.
(2) 바인딩 : 생성된 소켓을 특정 IP 주소와 포트 번호에 할당한다.
(3) 리스닝 및 연결 : 서버 소켓은 클라이언트의 연결 요청을 기다리며, 클라이언트 소켓은 서버에 연결을 요청한다.
(4) 데이터 송수신 : 연결이 성립되면 양측은 데이터를 주고받을 수 있다.
(5) 연결 종료 : 통신이 끝나면 소켓 연결을 종료한다.
