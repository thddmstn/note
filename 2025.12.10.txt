1. 참고사항
 1) 아스키코드 : 256개의 문자조합 제공 1바이트 문자 인코딩 방식, 영어권 0~127, 비영어권 128~255 코드 번호 사용


2. 수정자 (ignore case, global, multi line, single line)
 1) i :  ignore case 대소문자 무시

 2) g : global 일치하는 모든 문자열 검사
  (1) /regular expression/g  인경우 문서 전체에서 regular expression 를 찾는다.
  (2) g 해제시 : 첫번째 regular expression 만 찾는다.

 3) m : 앵커문자 모든 줄에 ^, $ 의 줄 구분 해제/설정    
  (1) g 없이 m 만 설정시 첫번째 값만 탐지한다. 줄의 위치는 상관 없이 첫번째 매칭된 내용이 탐지
  (2) 결국 m 수정자는  ^ 또는 $ 를 각 줄마다 표시하고 각각의 라인의 문자열을 검색 가능하다.
        ^.{3}    처음 3글자

 4) s : 메타문자  . 의 검사 범위에 줄바꿈 문자를 포함한다. 즉  $ 을 포함하여 검색한다.
  (1) regular(줄끝)expression(다음줄) 경우 s 설정시 $를 . 으로 표현 가능 (s 설정시 regular.expression 탐지됨)
  (2) Wireshark를 이용하여 테스트 가능 http.host contains "사이트"  ->  \r \n   ->  16진수로는 0D 0A
  (3) [Linux VIM 으로 테스트] :set hlsearch, :set nohlsearch, :nohlsearch
    - VIM 줄바꿈문자  :     /\n (set hlsearch, set list)
    -  VIM 싱글라인 모드 메타문자 :       \_
      hello, world 가 서로 다른 줄에 있을경우 /hello\_.*world 처럼 검색 가능(멀티라인 set magic)
      검색에서    .*  를 입력시 줄바꿈( $ ) 는 선택(흰색)되지 않는다,  (vim  ->  set list   ->    /    ->  .*   또는 \_.*    후 확인)
  (4) 자바스크립트는 싱글라인 수정자를 지원하지 않는다(현재)
